# Systematic Debugging Workflow

When encountering bugs or test failures, follow this systematic approach:

## 1. Reproduce
- Confirm the issue exists
- Identify the exact steps to trigger it
- Note any error messages or unexpected behavior

## 2. Isolate
- Narrow down the cause
- Check recent changes
- Use logging or debugging tools
- Identify the specific component or function

## 3. Fix
- Apply the solution
- Keep changes minimal and focused
- Add comments if the fix is non-obvious

## 4. Verify
- Test the fix works
- Run related tests
- Check for regressions
- Ensure the root cause is addressed
